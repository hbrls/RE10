# Task Dispatcher Workflow

用于在用户提出“需要拆解任务 / 分配执行者 / 给出分步提示词”时，先进行四类任务识别，再输出可执行的分派方案。

## 触发方式

- `/task-dispatcher`
- `帮我拆解这个任务`
- `这个任务该我做还是 AI 做？`
- `请把这件事分步骤并给提示词`

## 执行步骤

1. **任务分类**
   - 在以下四类中选择唯一类型：
     - 直接问题解决（Direct Problem Solving）
     - 直接输出生成（Direct Output Generation）
     - 协作问题解决（Collaborative Problem Solving）
     - 协作输出生成（Collaborative Output Generation）

2. **任务分析与分配**
   - 判断任务由谁主导：`人类主导` 或 `AI 主导`。
   - 解释分配依据（如：信息完整度、创造性要求、判断风险、迭代需求）。

3. **详细拆解**
   - 将任务拆为有依赖顺序的步骤。
   - 每一步必须给出：
     - 执行者（AI/人类）
     - 具体操作
     - 人类输入给 AI 的提示词
     - 人类需要完成的事项

4. **输出结构化结果**
   - 严格按模板输出，确保用户可直接照做。

## 任务类型判断标准

### 直接问题解决
- **特征**：有明确答案，信息相对完整，一次性交互可解决。
- **示例**：
  - 如何优化这段代码性能？
  - 这个 API 参数说明是什么？
  - 帮我找出这个 bug 的原因。

### 直接输出生成
- **特征**：需要较长输出，输入信息充足，可一次性完成。
- **示例**：
  - 根据大纲写一篇 2000 字技术文章。
  - 基于数据生成分析报告。
  - 根据要求写一个 Python 脚本。

### 协作问题解决
- **特征**：问题复杂，信息不完整，需要多轮探索。
- **示例**：
  - 帮我设计完整系统架构。
  - 机器学习模型表现不佳，帮我优化。
  - 想创业但不知道方向。

### 协作输出生成
- **特征**：需要较长输出但信息不足，需多轮完善。
- **示例**：
  - 帮我写商业计划书。
  - 设计完整用户手册。
  - 创作小说大纲和正文。

## 输出模板（仅 Markdown）

```markdown
任务名称：[任务名称]
任务类型：[直接问题解决/直接输出生成/协作问题解决/协作输出生成]
任务分配：[人类主导/AI主导]
为什么属于此类型：[解释为何任务属于该类型]
为什么要交给[人类/AI]：[解释分配理由]

### 任务拆解步骤：

**步骤1：[步骤名称]**
- 执行者：[AI/人类]
- 具体操作：[详细操作说明]
- 人类输入给AI的提示词：[根据任务类型优化的提示词]
- 人类需要做的事：[详细说明]

[继续列出后续步骤...]
```

## 提示词策略模板

### 直接问题解决类提示词
```text
请直接回答以下问题：[问题描述]
要求：
1. 提供准确、简洁的答案
2. 如有多个解决方案，请比较优劣
3. 给出可操作的建议
```

### 直接输出生成类提示词
```text
请根据以下要求生成[内容类型]：

背景信息：[背景]
具体要求：[详细要求]
输出格式：[格式要求]
篇幅要求：[字数/长度]
风格要求：[写作风格]

请生成完整的[内容类型]。
```

### 协作问题解决类提示词
```text
我需要解决以下复杂问题：[问题描述]

目前已了解的信息：[已知信息]
面临的挑战：[具体困难]

请帮我：
1. 分析问题的核心要素
2. 提出可能的解决方案方向
3. 识别需要补充的关键信息
4. 制定下一步行动计划
```

### 协作输出生成类提示词
```text
我将与你协作创建[内容类型]，这是一个需要多轮完善的过程。

初步想法：[初步概念]
核心要求：[基本要求]
目标受众：[使用对象]

让我们先从确定整体框架开始，然后逐步完善细节。
请告诉我需要补充哪些信息才能更好地开始。
```

## 注意事项

- 拆解逻辑清晰，步骤之间依赖明确。
- 提示词要具体、可操作，避免抽象空话。
- 复杂任务先搭框架，再细化执行。
- 优先发挥人类（创造力/判断力）与 AI（信息处理/模式识别）各自优势。
