# Retrospective Workflow

用于在会话结束时执行轻量复盘：保留原版能力的关键处理方式（Review / Extract / Stitch / Confirm），但不依赖脚本与 JSON，仅在对话中输出 Markdown。

## 触发方式

- `/retrospective`
- `复盘一下刚才的对话`
- `把本次会话做个复盘`
- `总结这次会话的得失`

## 执行步骤

1. **扫描本次会话（Review）**
   - 覆盖本次会话全流程：目标、上下文、关键决策、执行结果。
   - 标记正向信号与负向信号：哪些做法有效，哪些引发返工/误解/风险。
   - 对问题做简要归因（原因而非仅现象）。

2. **提取经验（Extract）**
   - 从对话中提取可复用经验，按 Markdown 小节组织：
     - 用户偏好（Preferences）
     - 修复与规避（Fixes）
     - 可复用提示/方法（Prompts / Playbook）

3. **生成可沉淀草案（Stitch Preview）**
   - 在对话中给出“可直接沉淀”的 Markdown 草案。
   - 仅给出建议落点（如 `SKILL.md` 或 `references/*.md`），不自动写入文件。

4. **确认是否沉淀（Confirm）**
   - 复盘末尾必须追加：
     - `以上是本次会话复盘，是否需要我将这些经验沉淀到指定文档？`
   - 未获用户明确同意前，禁止执行任何落盘写入。

## 输出模板（仅 Markdown）

```markdown
## 本次会话复盘

### 1. 背景与目标
- ...

### 2. 过程回顾
- ...

### 3. 做得好的点
- ...

### 4. 可以改进的点
- ...

### 5. 可沉淀经验草案
- ...

### 6. 确认
以上是本次会话复盘，是否需要我将这些经验沉淀到指定文档？
```

## 约束

- 仅使用 Markdown 输出，不输出 JSON。
- 复盘范围必须覆盖本次会话所有过程。
- 默认轻量，先事实后判断，避免主观泛化。
